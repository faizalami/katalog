Mengapa Alat ini Dibuat?

Pertumbuhan internet yang sangat pesat berdampak pada peningkatan jumlah kejahatan siber.

Intrusion Detection System (IDS) atau Sistem Deteksi Intrusi adalah salah satu solusi dalam memonitor masalah ini. Mata Garuda adalah salah satunya.

Mata Garuda menghasilkan data yang sangat banyak, sehingga visualisasi data tersebut dibutuhkan.

Visualisasi diperlukan agar dapat dilakukan analisis dengan cepat.

Visualisasi harus real-time agar analisis dapat dilakukan secepat mungkin.

Aplikasi ini juga harus mengelola data secara historis agar dapat dilakukan analisis serangan lebih lanjut.

Manfaat dan Kegunaan

Mengembangkan aplikasi yang memvisualisasikan serangan internet secara real-time.

Serangan internet ditampilkan dalam dimensi spasio-temporal.

Hasil dari proyek akhir ini digunakan untuk menampilkan peta serangan internet secara real-time pada aplikasi Mata Garuda, sehingga diharapkan agar dapat dilakukan analisis serangan dengan cepat dan efisien.

Hasil dari proyek akhir ini digunakan untuk menampilkan peta serangan internet secara real-time pada aplikasi Mata Garuda, sehingga diharapkan agar dapat dilakukan analisis serangan dengan cepat dan efisien.

Fitur

  • Memvisualisasikan serangan internet secara real-time.
  • Memvisualisasikan serangan internet secara historis – berdasarkan interval waktu tertentu (harian, mingguan, bulanan).
  • Visualisasi-visualisasi tersebut dalam dimensi spasio-temporal.

Metode / Desain Sistem

  • Data Collecting
  • Data serangan internet berasal dari log alert yang dihasilkan oleh sensor Snort. Data yang didapat dari log alert snort dikumpulkan oleh Apache Kafka malalui protocol MQTT.
  • Data Processing
  • Apache Spark digunakan untuk memproses data. Pada tahap ini, ditambahkan informasi berupa lokasi dari sumber dan tujuan dari serangan. Data lokasi didapatkan dari database GeoIP MaxMind. Data diproses dengan menggunakan Sliding Window Algorithm.
  • Data Storing
  • MongoDB menyimpan data yang telah diproses dari Spark. Fitur Change Stream digunakan agar aplikasi dapat mengakses perubahan data secara real-time.
  • Data visualization
  • Node.js pada sisi back-end mendapatkan notifikasi dari MongoDB kemudian mengirimkan data tersebut pada sisi front-end melalui Socket.io.Pada front-end data serangan ditampilkan menggunakan modul d3-geo dari D3.js.
