Mengapa Alat ini Dibuat?

Teknologi-teknologi yang digunakan dalam pengembangan aplikasi haruslah selalu mutakhir dan kualitas kode harus sangat diperhatikan untuk menjaga agar aplikasi yang dikembangkan dapat selalu diperbaharui. Kerap kali, perusahaan sering sekali memberikan deadline pengerjaan aplikasi dalam waktu yang sangat singkat. Singkatnya waktu pengerjaan mengakibatkan tidak selesainya proyek atau aplikasi selesai dengan tidak sempurna. Sering kali, dalam pengerjaannya programmer akan mengutamakan untuk mengejar target selesai secepat mungkin sehingga akan mengesampingkan kaidah-kaidah struktur kode yang baik.

Manfaat dan Kegunaan

Memberikan solusi baru untuk memudahkan para penyedia jasa pembuatan aplikasi dalam mempercepat penyelesaian proyek pembuatan aplikasi dengan pembuatan generator aplikasi web dengan gui builder dan autogenerate database sebagai sebuah Laravel package. Hasil generate merupakan suatu aplikasi untuk menjalankan operasi CRUD (Create, Read, Update, dan Delete) untuk tabel-tabel yang tidak berelasi pada database. Source code hasil generate menerapkan fitur-fitur framework Laravel, best practice, software design pattern dan clean code pada hasil generate sehingga pembuatan aplikasi menjadi cepat dan menghasilkan struktur kode yang ringkas, mudah pahami, dan mudah dikembangkan.

Fitur

  • Pages Manager untuk memanajemen dan mendesain aplikasi dengan drag and drop GUI Builder.
  • Project Configuration untuk mengubah tema dan layout aplikasi.
  • REST Manager untuk menentukan format response JSON pada REST API yang digenerate.
  • Database Manager untuk manajemen tabel-tabel database yang telah digeneate dan untuk generate dummy data.

Metode / Desain Sistem

  • User Interface merupakan layer input dari sistem Laravel-Mager. User Interface adalah tampilan antarmuka untuk digunakan oleh pengguna dalam memberikan input pada sistem. User interface yang disediakan akan dikelompokkan ke dalam empat kategori, yaitu Pages Manager, Project Configuration, REST Manager, dan Database Manager. Bagian User Interface ini memberikan output berupa file konfigurasi JSON.
  • Configuration reader adalah komponen untuk membaca file JSON Configuration untuk diterjemahkan ke dalam object-object atau array-array pada PHP. Untuk membangunnya dibutuhkan fungsi-fungsi JSON pada bahasa pemrograman php.
  • Kemampuan me-render view Laravel Blade dapat dimanfaatkan untuk melakukan generate, dengan melakukan render file template bertipe Laravel Blade yang digabungkan dengan hasil pembacaan file konfigurasi JSON ke dalam sebuah variable bertipe string.
  • Hasil render template dengan konfigurasi akan menghasilkan variabel bertipe string. Variabel ini akan disimpan ke dalam file berdasarkan jenis file sesuai dengan konfigurasi, misalkan konfigurasi yang dilakukan render adalah konfigurasi controller, maka file akan disimpan ke dalam direktori controller framework Laravel dan dengan tipe file PHP.
  • Beberapa konfigurasi membutuhkan penanganan khusus karena ada hubungannya dengan database. Karena ada hubungannya dengan objek-objek pada database, komponen ini menggunakan beberapa fitur PHP dan Laravel, yaitu PHP PDO , Laravel Artisan, dan Laravel Elequent untuk melakukan generate komponen-komponen database.
  • Output dari package Laravel-Mager berupa file-file source code framework Laravel dan komponen-komponen database. Komponen-komponen database berupa skema database, tabel dan data pada database.

Output

Halaman Pages Manager untuk memanajemen halaman-halaman aplikasi.

GUI Builder untuk mendesain aplikasi dengan drag and drop.

Project Configuration untuk mengubah tema dan layout aplikasi.

REST Manager untuk menentukan format response JSON dari REST API.

Database Manager untuk memanajemen tabel-tabel database dan generate dummy data.

Aplikasi siap pakai hasil dari generate.
