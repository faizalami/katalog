Mengapa Alat ini Dibuat?

Dalam rekayasa perangkat lunak, pengaplikasian design pattern dalam mengembangkan perangkat lunak dapat mempercepat proses pengembangan dan mengurangi dampak dari maintenance tidak terkecuali juga dalam mengembangkan automation testing. Selama ini quality assurance engineer sering kali menggunakan design pattern page object dibanding design pattern yang lain untuk mengembangkan automation testing. Penggunaan page object yang membuat objek pada aplikasi berbasis web menjadi objek lebih kecil lagi, menjadikan page object mudah untuk diimplementasikan dan mudah untuk dilakukan maintain. Akan tetapi dewasa ini design pattern screenplay juga menjadi design pattern yang tidak kalah populer di kalangan quality assurance engineer dalam mengembangkan automation testing. Pengimplementasian prinsip-prinsip rekayasa perangkat lunak yang baik seperti Single Responsibility Principle dan Open-Closed Principle pada design pattern screenplay, menjadikan quality assurance engineer lebih mudah dalam melakukan maintain dan dokumentasi.

Manfaat dan Kegunaan

Tujuan dari proyek akhir ini adalah menampilkan hasil analisis kualitas dari design pattern page object dan screenplay sehingga mampu menjadi patokan atau sumber referensi bagi quality assurance engineer dalam memilih design pattern yang akan digunakan untuk mengoptimalkan automation testing mereka. Dengan melihat hasil penelitian ini, quality assurance engineer dapat mengetahui nilai Functionality, Reusability, Flexibility, Understandability, Effectiveness, dan Extendibility dari kedua design pattern tersebut sebagai referensi mereka untuk menentukan design pattern yang akan digunakan

Fitur

  • Nilai Functionality, Reusability, Flexibility, Understandability, Effectiveness, dan Extendibility dari design pattern Page Object dan Screenplay

Metode / Desain Sistem

  • Input:
  • Test case dijalankan pada website yang hendak diuji
  • Proses:
  • Pengimplementasian design pattern page object dan screenplay pada automation testing
  • Design pattern page object dan screenplay diintegrasikan pada framework terkait
  • Perhitungan nilai QMOOD dari masing masing design pattern
  • Output:
  • Test report dari automation testing
  • Hasil perhitungan menggunakan QMOOD

Output

Hasil test report pada salah satu tes skenario dengan mengimplementasikan design pattern Page Object. Tes skenario pada gambar tersebut merupakan skenario “Checkout with Default Address”, dimana user mengonfirmasi pemesanan dengan alamat yang sudah ada. Dari gambar tersebut, dapat diketahui bahwa hasil test report pada skenario ini adalah success.

Hasil test report pada salah satu tes skenario dengan mengimplementasikan design pattern Screenplay. Tes skenario pada gambar tersebut merupakan skenario “Checkout with Default Address”, dimana user mengonfirmasi pemesanan dengan alamat yang sudah ada. Dari gambar tersebut, dapat diketahui bahwa hasil test report pada skenario ini adalah success.

Grafik komparasi hasil kalkulasi menggunakan metode QMOOD. Dari grafik tersebut dapat diketahui bahwa design pattern page object memiliki kualitas yang lebih baik dibanding design pattern screenplay dalam segi functionality dan effeciveness. Sedangkan design pattern screenplay memiliki kualitas yang lebih baik dibandign design pattern page object dalam segi reusability, flexibility, understandability, dan extendibility.
