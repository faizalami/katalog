Mengapa Alat ini Dibuat?

Code smell adalah karakteristik struktural perangkat lunak yang dapat menunjukkan masalah pada desain kode dan membuat perangkat lunak menjadi sulit untuk dikembangkan dan dipelihara. Lazy Class merupakan kelas yang memiliki fungsionalitas yang rendah atau hampir tidak diperlukan. Dihilangkannya lazy class tersebut akan mengurangi kompleksitas program sehingga lebih mudah dipahami dan dikembangkan. Namun, code smell merupakan salah satu faktor yang sulit untuk diidentifikasi karena perbedaan pengalaman dari setiap programmer.

Agar ancaman-ancaman yang dapat disebabkan oleh code smell dapat dihindari, maka perlu dilakukan refactoring pada code smell tersebut. Namun, melakukan refactoring pada kode bukanlah proses yang sederhana. programmer dituntut untuk memiliki pengetahuan dan pengalaman dalam mengidentifikasi dan menangani code smell tersebut. Terlebih lagi proses refactoring yang dilakukan secara manual akan meningkatkan biaya pemeliharaan dari segi waktu dan usaha.

Manfaat dan Kegunaan

Penelitian proyek akhir ini ditujukan untuk membuat sebuah automated tool dalam bentuk library yang dapat menganalisa kode dalam bahasa pemrograman Java. Tool tersebut dapat mendeteksi adanya code smell dengan lazy class dan melakukan refactoring secara otomatis pada code smell tersebut. Sehingga, kedepannya tool ini dapat membantu para programmer dalam mengembangkan dan memelihara perangkat lunak, terlebih dalam menjaga dan meningkatkan kualitas desain kode sekaligus menghemat waktu dan usaha.

Fitur

  • Modul Deteksi Otomatis :
  • Deteksi berkas-berkas kode program
  • Deteksi kelas pada berkas kode program
  • Deteksi jumlah method pada kelas
  • Deteksi jumlah atribut pada kelas
  • Deteksi kelas-kelas yang terindikasi lazy class code smell
  • Modul Refactoring Otomatis :
  • Mendapatkan kandidat kelas target
  • Memeriksa hubungan hirarki lazy class
  • Menghitung jumlah akses resource dari lazy class
  • Memberikan nilai pada kandidat kelas
  • Mendapatkan kelas target terbaik
  • Memindahkan isi kelas lazy class ke kelas target.

Metode / Desain Sistem

  • Input :
  • file/folder path dari project yang menggunakan bahasa permrograman java
  • Proses :
  • Mendeteksi file-file java pada project secara otomatis
  • Menganalisa komponen dan struktur kelas java secara otomatis
  • Menghitung software metric pada tiap kelas yaitu NOM(Number of Method) dan NOF(Number of Field) secara otomatis
  • Mendeteksi kelas yang terindikasi code smell lazy class secara otomatis
  • Melakukan refactor pada kelas yang bermasalah secara otomatis
  • Output:
  • Project java yang berhasil direfactor secara otomatis dan bersih dari code smell lazy class

Output

output dari deteksi file dimana deteksi file dilakukan dengan menginputkan path suatu project untuk kemudian dideteksi file-file java pada project tersebut. Dari hasil yang didapat, tool mampu mendeteksi file dengan akurat.

sistem mampu mendeteksi kelas java sebanyak file java yang terdeteksi. Namun, dalam beberapa kasus pendeteksian kelas dapat menghasilkan jumlah yang berbeda dengan jumlah file yang terdeteksi. Perbedaan jumlah tersebut bisa diakibatkan oleh file java yang tidak memiliki kelas di dalamnya maupun file java yang memiliki lebih dari satu kelas di dalamnya.

tool mampu mendeteksi kelas mana saja yang tergolong lazy class secara akurat dengan mempertimbangkan jumlah method dan attribute dari setiap kelas yang kurang dari threshold. Pendeteksian dapat dilakukan karena tool berhasil melakukan perhitungan software metric pada ClassModel sebelumnya.

proses pendeteksian secara manual membutuhkan waktu yang jauh lebih lama dibanding proses pendeteksian yang dilakukan menggunakan tool. Penggunaan Tool dapat meningkatkan proses deteksi hingga 53 kali lipat lebih cepat dibandingkan deteksi secara manual.

nilai setiap kelas pada project yang dihitung dengan menjumlahkan banyaknya setiap kelas mengakses method & field dari lazy class.

tool berhasil mendapatkan kelas target yang akan dijadikan sebagai kelas untuk menampung konten dari kelas lazy class.

sistem berhasil melakukan proses akhir dari refactoring dengan mengubah file secara langsung. Sistem berhasil mengahapus file lazy class dan mengganti file baru untuk kelas target dengan cara menghapusnya dan membuat file baru.

proses refactoring secara manual membutuhkan waktu yang jauh lebih lama dibanding proses pendeteksian yang dilakukan menggunakan tool. penggunaan ARETO dapat meningkatkan proses refactoring hingga 66 kali lipat lebih cepat dibandingkan deteksi secara manual
